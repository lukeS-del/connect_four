# Четыре в ряд — сервер (Node.js + Socket.IO)

Полноценный мультиплеер: сервер хранит состояние комнаты и авторитетно считает ходы,
клиенты подключаются через WebSocket (Socket.IO) и мгновенно видят ходы друг друга.

## Структура проекта

```
connect-four-server/
├── index.js          # сервер (Express + Socket.IO)
├── package.json
└── public/
    ├── index.html
    ├── style.css
    └── client.js      # клиентская логика на сокетах
```

## 1. Проверка локально (по желанию)

```bash
cd connect-four-server
npm install
npm start
```

Открой `http://localhost:3000` в двух вкладках — они должны видеть ходы друг друга.

## 2. Деплой на Render.com

### Шаг 1. Залей код на GitHub
1. Создай новый репозиторий на github.com (например `connect-four`)
2. В папке `connect-four-server` выполни:
   ```bash
   git init
   git add .
   git commit -m "Connect Four server"
   git branch -M main
   git remote add origin https://github.com/ТВОЙ_ЛОГИН/connect-four.git
   git push -u origin main
   ```

### Шаг 2. Создай Web Service на Render
1. Зайди на [render.com](https://render.com), зарегистрируйся (можно через GitHub)
2. **New +** → **Web Service**
3. Подключи свой GitHub-репозиторий `connect-four`
4. Настройки:
   - **Name**: `connect-four` (или любое)
   - **Region**: ближайший к тебе (например Frankfurt)
   - **Branch**: `main`
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
   - **Instance Type**: `Free` (для теста; см. примечание про сон ниже)
5. Нажми **Create Web Service** — Render соберёт и запустит проект (1–3 минуты)

После деплоя ты получишь адрес вида:
```
https://connect-four-xxxx.onrender.com
```
Уже сейчас по этой ссылке можно играть вдвоём с любого устройства.

### Шаг 3. Подключи свой домен
1. Купи домен у любого регистратора (Reg.ru, Namecheap, Cloudflare и т.д.), если ещё не куплен
2. В Render: открой свой сервис → **Settings** → **Custom Domains** → **Add Custom Domain**
3. Введи свой домен (например `connect4.mysite.ru`)
4. Render покажет DNS-запись, которую нужно добавить:
   - Для поддомена — обычно **CNAME**, указывающий на `connect-four-xxxx.onrender.com`
   - Для корневого домена (`mysite.ru` без поддомена) — Render выдаст **A-запись**
5. Зайди в панель управления DNS у регистратора домена и добавь эту запись
6. Подожди 10 минут – несколько часов (обновление DNS), Render автоматически выпустит бесплатный SSL-сертификат (https)

Готово — сайт открывается по твоему домену.

## Важные нюансы

- **Бесплатный тариф Render "засыпает"** после ~15 минут без запросов и "просыпается" 30–50 секунд при первом заходе. Для игры с друзьями это заметная задержка при первом подключении. Если это критично — платный тариф Starter ($7/мес) держит сервер постоянно активным.
- **Состояние комнат хранится в памяти сервера** — при перезапуске сервера (деплой новой версии, сон/пробуждение на некоторых тарифах) все текущие комнаты сбрасываются. Для продакшена с большой нагрузкой стоит вынести состояние в Redis, но для игры с друзьями этого не требуется.
- **При обновлении страницы** игрок теряет свою роль (1 или 2) в комнате — соединение считается новым. При желании можно доработать: сохранять код комнаты и роль в `localStorage` на клиенте и переподключаться автоматически.
- Комнаты, к которым не было обращений 6+ часов, автоматически удаляются с сервера (см. `ROOM_TTL_MS` в `index.js`).
